# Locked dependency tree for the Marsh orchestrator.
#
# The orchestrator's ONLY direct dependency is daytona-sdk; everything else here
# is its locked transitive closure. This is a full lockfile, not a loose pin, for
# two reasons: (1) reproducible deploys on the orchestrator host; (2) accurate
# CI dep-scanning — osv-scanner's
# transitive resolver otherwise picks ancient minimum-satisfying versions (e.g.
# requests 2.9.2) that are never actually installed and trip --fail-on high on
# phantom CVEs. Pinning the real tree makes the scan reflect reality.
#
# Regenerate after bumping daytona-sdk:
#   python3 -m venv /tmp/v && /tmp/v/bin/pip install 'daytona-sdk==<new>' \
#     && /tmp/v/bin/pip freeze | grep -viE '^(pip|setuptools|wheel)==' \
#     > orchestrator/requirements.txt
